# 讲话稿与发言提纲

## 先确认场景
- 谁讲：身份、职责、语气边界。
- 对谁讲：上级、同级、下级、客户、公众或内部团队。
- 为什么讲：动员、汇报、总结、部署、感谢、表态或交流。
- 讲多久：篇幅决定结构密度。

## 常用结构
- 开场：点明场景和主题，不铺垫过长。
- 主体：三到四个层次，每层先观点后材料。
- 措施：写清重点任务、方法和要求。
- 结尾：回扣主题，给出号召、承诺或下一步。

## 语言控制
- 口语可讲但不随意，书面稳重但不僵硬。
- 避免空泛排比堆叠；每个大标题最好有实际信息。
- 不代替用户编造职务、批示、成绩、政策提法或敏感口径。

## 来源索引

- 《中华人民共和国国家通用语言文字法》：规范汉字和标准化表达。
