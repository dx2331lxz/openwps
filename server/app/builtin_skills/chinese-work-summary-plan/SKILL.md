# 工作总结与计划

## 总结框架
- 背景和目标：说明周期、职责范围、核心目标。
- 主要工作：按任务线、项目线或时间线组织，不罗列流水账。
- 成效数据：尽量用可核验数据、案例和对比说明。
- 问题不足：写事实和原因，不泛泛自责。
- 下一步：目标、措施、责任、节点、风险和保障。

## 计划框架
- 目标：具体、可衡量、可验收。
- 重点任务：每项写清动作、产出、时间和协同方。
- 资源保障：人员、预算、制度、工具和沟通机制。
- 风险预案：列出关键风险和应对策略。

## 改写要求
- 保留用户事实，不编造业绩、数据、荣誉和政策依据。
- 把“做了很多工作”改成“围绕什么目标、采取什么动作、形成什么结果”。
- 减少套话，增强层次标题的信息量。
