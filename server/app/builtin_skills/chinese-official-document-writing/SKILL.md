# 中文公文写作

## 工作方式
- 先判断文种和行文关系：上行用请示、报告；下行常用通知、批复、决定、通报；平行或不相隶属机关多用函。
- 先问清或从上下文提取：发文机关、主送机关、事项依据、办理要求、期限、联系人、附件、是否涉密或紧急。
- 起草时按“依据/背景 -> 事项/问题 -> 要求/措施 -> 期限/责任”的顺序组织，保持主题集中、结构严谨、文字精练。
- 如用户没有给足事实，不要编造政策、批复意见、会议决定、数据或机关名称；用待补充项标出缺口。
- 输出前做审核：文种是否正确，标题是否含发文机关、事由、文种，主送机关是否匹配，附件和落款是否一致，时间、数字、人名、地名、引文是否准确。

## 常见文种提示
- 通知：说明发布、传达、执行或周知事项，重点写清执行对象、事项、期限和要求。
- 请示：一文一事，写明请求事项和理由，结尾用请求批示类表达，不同时抄送下级机关。
- 报告：汇报工作、反映情况或回复询问，不夹带请示事项。
- 函：用于商洽、询问、答复或请求批准，语气平实、边界清楚。
- 批复：先对应来文，再明确同意、原则同意或不同意，并列出执行要求。
- 纪要：记录会议主要情况和议定事项，不写成流水账。

## 语言风格
- 优先使用准确、平实、克制的书面语，避免口号堆叠、空泛形容和营销语。
- 政策措施要可执行：责任主体、办理路径、时间节点、验收方式尽量明确。
- 涉及法规、政策、机构职责时，提醒用户核对正式依据和最新口径。
