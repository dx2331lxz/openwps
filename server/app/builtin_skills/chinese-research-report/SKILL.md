# 中文调研报告

## 工作流
- 先确认调研对象、范围、样本、时间、数据来源和目标读者。
- 用“现状 -> 问题 -> 原因 -> 对策”的主线组织，避免材料堆砌。
- 事实、观点、建议分开写；每个判断尽量对应证据。
- 对策建议要写清责任主体、实施路径、资源条件、预期效果和风险。

## 常用结构
- 摘要：一句话结论 + 关键发现 + 核心建议。
- 背景与方法：说明调研范围和资料来源。
- 主要发现：按主题分组，每组先结论后证据。
- 问题及原因：区分表象问题、制度原因、资源原因、执行原因。
- 建议：按短期可做、中期优化、长期机制分层。

## 风险提示
- 不替用户虚构访谈、问卷、统计结果或政策依据。
- 数据不足时明确写“基于现有材料的初步判断”。
- 涉及敏感评价时保持客观、克制、可证据化。
